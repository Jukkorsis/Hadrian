State

Service Version

API
The version of the service's API.
Impl
The current version of the software that implements the service.
Status
The life cycle of this version of the service:
Proposed, The service has been proposed and is under development, but is not yet in production.
Active, The service is in production and is actively maintained.
Live, The service is in production but is not actively being maintained.
Retiring, The service is in production but the owning team is in the process of turning off the service.
Retired, The service is no longer in production.
Operations
A list of operations/URLs that can be performed on the versions, such as health, version, availability, etc.
Scope
Scope defines the nature of service's dependence on another service:
Sync Required, To complete critical functions this service makes synchronous calls to the dependant service.
Async Required, To complete critical functions this service makes direct async calls to the dependant service.
Indirect Required, To complete critical functions this service makes indirect async calls via a queue to the dependant service.
Support, In the background this service makes support calls to the dependant service, for example logging or metrics.
